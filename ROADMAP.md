# k10s Roadmap

## Product Identity

> **k10s는 쿠버네티스를 몰라도 쓸 수 있는 원터치 개발환경 런처입니다.**
>
> 여러 클러스터에 흩어진 내부 엔드포인트(DB, Redis, API 등)를 포트포워딩으로 로컬에 꺼내
> 개발환경을 구성하는 과정을 **극단적으로 간소화**합니다.
>
> YAML을 직접 편집할 필요 없이, TUI를 사용하다 보면 히스토리가 자동으로 쌓이고,
> 자주 쓰는 것은 즐겨찾기로, 즐겨찾기를 묶으면 세트로 — 모든 구성이 **TUI 안에서 직관적으로** 완성됩니다.

### 설계 원칙

1. **Zero Config**: 처음 설치 후 바로 실행. 설정 파일을 만들 필요 없음.
2. **쓰면서 쌓이는 구성**: 수동으로 YAML을 편집하지 않아도, TUI를 쓰다 보면 히스토리와 프리셋이 자동으로 축적됨.
3. **TUI-First 관리**: 즐겨찾기 등록, 세트 구성, 삭제 — 모든 설정 관리를 TUI 메뉴 안에서 해결.
4. **k8s 무지렁이도 OK**: 쿠버네티스 용어를 최소화하고, 모르는 사람도 화면 안내만 따라가면 사용 가능.

### k9s와의 차별점

| | k9s | k10s |
|---|---|---|
| **대상** | kubectl에 익숙한 엔지니어 | k8s를 잘 몰라도 쓸 수 있는 누구나 |
| **초점** | 단일 클러스터 리소스 탐색 | 멀티 클러스터 포트포워드 개발환경 |
| **설정** | kubeconfig 직접 관리 | TUI가 자동으로 쌓아주는 히스토리/즐겨찾기 |
| **핵심 가치** | 탐색(Explore) | 원터치 실행(One-Touch Launch) |

## Phase 2: PF 세트 (개발환경 원터치 구성)

**목표**: 여러 포트포워드를 "세트"로 묶어 한 번에 불러오는 개발환경 원터치 실행.

### 사용 시나리오

```
k10s 실행
  → 포트포워드 관리 진입
  → 평소처럼 Redis PF 생성 (히스토리에 자동 저장)
  → PostgreSQL PF 생성 (히스토리에 자동 저장)
  → API Server PF 생성 (히스토리에 자동 저장)
  → [s] "현재 활성 PF를 세트로 저장" 선택
  → 세트 이름 입력: "백엔드 개발" → 저장 완료

다음날 출근
  → k10s 실행
  → 포트포워드 관리 진입
  → "백엔드 개발" 세트 선택 → PF 3개 일괄 실행 ✅
```

### 핵심: YAML 편집 없는 세트 구성

| 기능 | 수행 방법 |
|---|---|
| 세트 생성 | TUI에서 PF를 띄운 후 `[s]` → 현재 활성 PF를 세트로 저장 |
| 세트 실행 | 포트포워드 매니저에서 세트 목록 표시 → 선택 시 일괄 실행 |
| 세트 종료 | 매니저에서 세트 단위로 일괄 종료 |
| 세트 편집 | TUI 안에서 세트에 PF 추가/제거 |
| 세트 삭제 | `ctrl+d`로 삭제 (기존 패턴과 일관성 유지) |

### 자동으로 쌓이는 구성

- 포트포워드를 한 번이라도 실행하면 → **히스토리에 자동 기록**
- 히스토리에서 자주 쓰는 PF를 `[f]` → **프리셋(즐겨찾기)으로 승격**
- 프리셋 여러 개를 묶어 `[s]` → **세트로 구성**
- 모든 과정이 TUI 메뉴 안에서 완결, config.yaml은 백그라운드에서 자동 저장

```
히스토리 (자동)  →  프리셋 (즐겨찾기 등록)  →  세트 (묶어서 원터치)
   [자동 축적]         [f] 키로 승격            [s] 키로 그룹화
```

## Phase 3: 원터치 런치 화면

**목표**: k10s를 켜자마자 첫 화면에서 최근 사용한 세트를 바로 실행.

### 주요 기능

- **퀵 런치**: 첫 화면 상단에 최근/자주 사용한 세트 표시
- **원터치 실행**: 세트 옆 단축키(`1-9`) 누르면 클러스터 선택 과정 없이 즉시 PF 가동
- **상태 요약**: 현재 활성 PF 개수 / 세트 상태를 한눈에 표시
- **사용 빈도 자동 정렬**: 많이 쓰는 세트가 상단으로
